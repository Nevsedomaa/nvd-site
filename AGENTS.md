# AGENTS.md

Короткая памятка для будущих правок проекта `nvd-site`.

## Проект

Статический сайт группы «Не все дома»: обычные HTML/CSS/JS без сборщика.

Основные страницы:

- `index.html` - главная.
- `releases.html` - список релизов.
- `album.html?id=...` - шаблонная страница релиза.
- `lyrics.html` - список текстов с поиском и фильтром по альбомам.
- `lyric.html?id=...` - шаблонная страница полного текста песни.
- `merch.html` - страница мерча.

## Общие правила

- Не добавлять сборщик и фреймворк без явной просьбы.
- Данные держать в JS-файлах, а страницы рендерить из них.
- Все изображения должны быть `draggable="false"` или блокироваться через общий `dragstart` в `js/script.js`.
- Favicon уже подключен на HTML-страницах через `assets/images/logo.png`.
- Не ломать темную визуальную систему сайта: `#0F1217`, `#161B22`, `#2B313A`, акцент `#464D6F`.
- Карточки и элементы держать с радиусом около `8px`.
- На главной блок `Строки` должен быть компактным: цитаты в 1-2 строки, не в 3.

## Релизы, треки, тексты

Главный источник данных: `js/releases-data.js`.

В нем лежат:

- релизы;
- треки;
- длительности;
- пути к mp3;
- короткие цитаты `lyricsExcerpt`;
- полные тексты `lyrics`.

При добавлении mp3 не нужно менять логику сайта: положить файл в `assets/audio` с именем из поля `audio`.

Список ожидаемых имен mp3 есть в `assets/audio/README.md`.

У каждого трека должен быть уникальный `id`, потому что:

- `album.html` строит ссылки на `lyric.html?id=track-id`;
- `lyrics.html` и `lyric.html` ищут текст по этому `id`;
- цитаты на главной тоже ведут на `lyric.html?id=...`.

Тексты альбома `Местоимение` частично перенесены из Word-файла. Текст песни `Прощание` в исходном Word-файле найден не был, поэтому там пока заготовка.

Из `NVD_teksty_2_ispravleno.docx` добавлены тексты: `putevoditel`, `prosti`, `svet-i-tma`, `dozhd`, `voshod`, `strah-temnoty`, `nvd`, `kuda-bezhat-za-schastem`.

## Страница текстов

Файлы:

- `lyrics.html`
- `lyric.html`
- `css/lyrics.css`
- `js/lyrics.js`
- `js/lyric.js`

На `lyrics.html` есть:

- поиск по названию/альбому/цитате;
- белый крестик очистки поиска;
- фильтр по альбомам.

Не возвращать нативный градиентный крестик браузера в поиске.

На `lyric.html` полные тексты должны выглядеть как загруженный текст альбома `Местоимение`: строки внутри куплета идут плотно, куплеты разделяются настоящими абзацами. Не возвращать один общий `<pre>` для текста песни; `js/lyric.js` рендерит абзацы по пустым строкам из `lyrics`.

На `lyrics.html` карточки берут цитату из `lyricsExcerpt`. Это должна быть вручную выбранная сильная строка из песни, а не автоматически первая строка текста.
`lyricsExcerpt` не должен заканчиваться запятой или выглядеть как оборванная строка; выбирай законченную фразу.

## Плеер альбома

Файлы:

- `album.html`
- `css/album.css`
- `js/album.js`

При раскрытии трека на странице альбома используется кастомный кассетный плеер, а не нативный `audio controls`.

Ассеты плеера:

- `assets/images/wheel-1.png` - левая бабина;
- `assets/images/wheel-2.png` - правая бабина;
- `assets/images/Volume.png` - крутилка громкости.

Правила:

- скрытый `<audio>` остается источником воспроизведения;
- бабины вращаются только когда музыка играет;
- верхние точки показывают текущую громкость;
- точки громкости находятся над центральной консолью, а не внутри нее;
- крутилка громкости управляет `audio.volume`;
- прогресс трека можно перематывать через кастомный range;
- внутренняя сетка плеера должна быть резиновой от ширины карточки, потому что колонка альбома узкая даже на десктопе;
- раскрытый плеер не должен иметь левый отступ под названием трека: он идет почти на всю ширину карточки с равными боковыми отступами;
- при переходе на другую страницу треков текущий звук останавливается.

## Мерч

Файлы:

- `merch.html`
- `css/merch.css`
- `js/merch-data.js`
- `js/merch-page.js`
- `js/order-config.js`

Данные товаров живут в `js/merch-data.js`.

У каждого товара:

- уникальный `id`;
- `title`;
- `description`;
- `price`;
- массив `images` из 3 фото.

Фото мерча должны меняться только при наведении на карточку. Без автосмены, когда пользователь не навел мышку.

При уходе мышки карточка возвращает первое фото.

Все карточки мерча должны быть одинаковой высоты, кнопки «Заказать» на одном уровне.

Рекомендованный нейминг фото: `assets/images/merch/<product-id>-1.jpg`, `...-2.jpg`, `...-3.jpg`.

Подсказка лежит в `assets/images/merch/README.md`.

Форма заказа уже реализована:

- открывается по кнопке «Заказать» относительно выбранного товара;
- затемняет фон;
- закрывается по клику на фон, по крестику и по `Escape`;
- валидирует имя, контакт, размер для футболок и согласие с политикой;
- для CD/дисков поле размера скрывается;
- ссылка политики ведет на Google Docs пользователя.

По последней просьбе пользователя Telegram token и chat id лежат прямо в `js/order-config.js`, без шифрования. Это небезопасно для публичного сайта, но пользователь явно попросил оставить ключи в коде.

Supabase для мерча подключен напрямую из фронта:

- url: `https://cbfkvlbaqofrxjgymjhj.supabase.co`;
- table: `merch`;
- ключ лежит в `js/order-config.js`;
- `js/merch-page.js` перед отправкой мапит внутренние поля формы в реальные колонки таблицы.

Колонки Supabase-таблицы `merch`, которые сейчас ожидает фронт:

- `id`
- `Product`
- `Prise`
- `Name user`
- `Сontact` - первая буква кириллическая `С`, не латинская `C`;
- `Size`
- `Comment`
- `Time`

Если в Supabase включен RLS, нужна insert policy для anon/publishable key, иначе браузер получит ошибку при записи.
Для `sb_publishable_...` ключа `js/merch-page.js` отправляет только `apikey`; `Authorization: Bearer ...` добавляется только для JWT/anon-ключа вида `eyJ...`.
На 2026-05-07 выяснилось, что колонка контакта в Supabase названа `Сontact`, где первая буква кириллическая `С`, а не латинская `C`. В `js/order-config.js` это задано как `"\u0421ontact"`. Лучше позже переименовать колонку в нормальный латинский `"Contact"` и поменять маппинг обратно.

`js/merch-page.js` отправляет заявки напрямую в Telegram API через `botToken + chatId` из фронта. Прокси/локальный сервер для этой отправки удалены.
